### 1. 电机驱动模块 (SimpleFOC Mini / DRV8313)

*注意：PWM 引脚对应 TIM1 的三个通道。*

| STM32 引脚 | 功能定义 | 连接到驱动板 (SimpleFOC Mini) | 备注 |
| --- | --- | --- | --- |
| **PA8** | TIM1_CH1 | **IN1** | U 相 PWM |
| **PA9** | TIM1_CH2 | **IN2** | V 相 PWM |
| **PA10** | TIM1_CH3 | **IN3** | W 相 PWM |
| **PA11** | GPIO_Output | **EN** (Enable) | **使能引脚** (注意修改代码定义) |
| **GND** | 地 | **GND** | **必须共地！** |
| - | - | **VM / VCC** | 接 12V 电源 (不要接 STM32!) |

> **⚠️ 代码修改提醒**：
> 你的图片中将 **PA11** 配置为了 GPIO 输出（即使能）。请检查你的 `main.h` 或 `drv_bldc_power.c`，确保宏定义匹配：
> `#define BSP_GPIO_MOTOR_EN  GPIO_PIN_11`

### ✅ 接线检查清单 (Checklist)

1. **共地 (Common Ground)**：这是最重要的一点。STM32 的 GND、驱动板的 GND、AS5600 的 GND、USB转串口的 GND，这四者必须最终连在一起。
2. **Enable 引脚**：图片显示是 **PA11**。请务必确认你的杜邦线是从 PA11 接到了驱动板的 EN 脚。
3. **电源电压**：
* 驱动板 VM 接 12V。
* STM32 和 AS5600 接 3.3V (或通过 USB 5V 经 LDO 降压)。
* **千万不要把 12V 接到 STM32 的任何引脚上，否则瞬间烧毁。**


4. **I2C 上拉**：如果 AS5600 读取不稳定，检查 PB8/PB9 是否在模块上有上拉电阻（通常模块自带，如果是裸芯片则需要加 4.7k 电阻到 3.3V）。

### 26.2.27

**目标平台**: STM32F103C8T6 (无FPU) + SimpleFOC Mini驱动板 + AS5600编码器 + 2804云台电机

**当前状态**: ✅ 所有问题已解决。系统在速度闭环模式下运行稳定。

Issue 1：进入闭环瞬间触发 RTOS HardFault (一发指令即死机)
- 现象: 发送 c (闭环) 或 t (开环) 指令后，MCU 仅打印一行遥测日志便瞬间死机卡死，对后续指令无反应。
- 根因: LogTask 任务堆栈溢出。CubeMX 默认分配的 128 字 (Words) 堆栈过小。在无硬件 FPU 的 MCU 上，调用处理 %f 浮点格式化的 vsnprintf 函数时，局部变量瞬间耗尽了任务栈，破坏了 RTOS 调度器。
- 修复: 
  1. 在 FreeRTOS 配置中将 LogTask 的堆栈大小 (Stack Size) 增加到 512。

Issue 2：PWM 幽灵断电 (脑电波正常，肌肉瘫痪)
- 现象: 软件运行完美，日志持续刷新，但电机没有任何保持力矩，一动不动。
- 根因: 安全机制导致的硬件锁死。BSP_PWM_StopAll() 在停止时正确清零了高级定时器的 MOE (主输出使能) 位；但后续调用 DrvBldc_Enable(true) 试图恢复电机时，仅更新了占空比寄存器，未重新置位 MOE，导致物理引脚持续处于高阻态。
- 修复: 在 DrvBldc_Enable() 的开启分支中，补充调用 BSP_PWM_Init()，以唤醒硬件并重新激活 MOE 引脚输出。

Issue 3：“蚊子腿”级别的驱动电压 (数值缩放错误)
- 现象: 硬件使能后，电机依然无法转动，甚至无法克服轴承的静态摩擦力。
- 根因: FOC 算法底层采用 Q15 定点数格式 (满量程 32767 = 100% 占空比)。此前代码中设定的测试电压 Uq = 400，换算后仅占 1.2% 的占空比 (在 12V 供电下仅有约 0.14V)，驱动力微乎其微。
- 修复: 在 test_console.c 中全面放大测试基准参数。将零点对齐电压修改为 4000，开环转矩模式电压修改为 3000。

Issue 4：正反馈死锁 与 PID 积分限幅过小
- 现象: 在 t 模式下，电机暴力锁死在固定角度（如 96°）；在 c 模式下，即便速度误差很大，PID 输出的 Uq 依然卡死在 195 无法上升。
- 根因A (锁死): 电机 UVW 物理相线产生的旋转磁场方向，与 AS5600 编码器的读数增加方向相反，导致坐标变换形成正反馈互相卡死。
- 根因B (限幅): PID 的积分累加器上限 (max_int) 初始化为 50000。由于算法内部 Q8.8 格式的右移 8 位操作 (>> 8)，导致最终 Q15 格式的实际输出上限被死死卡在了 195 (50000 / 256)，系统被严重“限流”。
- 修复: 
  1. 硬件调整: 随意对调电机的任意两根相线 (如 U 和 V) 以反转磁场方向。
  2. 软件调整: 将 AlgPid_SetGains 中的积分上限参数从 50000 暴力调大至 2000000，释放积分器的输出空间 (最高可达约 7812 的 Q15 电压)。
