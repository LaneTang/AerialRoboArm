## 🗺️ 下一步工程动作设计简报 (Next Steps Design Briefs)

为了完成“从 Homing 点出发，闭环控制到 Grab 点”的最终演示闭环，我们将在 `app_testbench.c` 中扩展以下槽位：

#### **1. Slot 5: 执行器实机标定测试 (Actuator Calibration)**
* **Layer**: L4 (Testbench Logic)
* **目标**: 建立舵机控制信号与末端执行器（夹爪与横滚）实际物理位置的绝对映射。
* **行为**:
    * 挂载 `mod_actuator.h` 及底层 PWM 驱动。
    * 用户通过终端热键切换控制对象（夹爪/姿态舵机），并通过 `+/-` 微调 PWM 脉宽。
    * **容错策略**: 利用 L2 层物理边界检测，探测并记录夹爪张开/闭合的最大安全角度，反馈至标定宏以防堵转烧毁。

#### **2. Slot 6: 自动抓取业务流预演 (Mock Auto Sequence)**
* **Layer**: L4 (System Orchestration FSM)
* **目标**: 验证 L4 级机电联控的防抖与收敛逻辑。
* **行为**:
    * 状态机监听 `TaskRc`，在 AUTO 模式且 CH3=100% 时触发联动。
    * 按照 `折叠臂默认位置(+15°) -> 伸出(+120°) 并行 夹爪张开/姿态转正 -> 闭环误差收敛验证 -> 执行抓合 -> 回收(+30°)` 的时序流转。
    * **容错策略**: 在 `HOLD_VERIFY` 状态中引入防抖计数器，若闭环目标误差无法连续 2 秒维持在 3° 内，则拒绝执行夹取操作；任何阶段触发 ESTOP 均瞬间退回安全闲置模式。
