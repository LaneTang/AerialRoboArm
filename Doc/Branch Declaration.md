# 🚀 Feature Branch: `demo_v3`

**RC Controller Integration & Architecture Refactoring (遥控器模块与架构重构)**

## 📑 概述 (Overview)

本分支致力于将 RC 遥控器（基于 ELRS/CRSF 协议）控制模块无缝整合至 ARA 机械臂多智能体系统（Multi-Agent System）中。
我们摒弃了最初的裸机阻塞式演示代码，严格按照 **ARA L1~L5 分层架构规约** 进行了全面重构，彻底消除了硬件依赖越级、浮点数运算负担以及多任务数据撕裂的风险，为机械臂的“自动/手动”模式平滑切换奠定了坚实的安全基础。

## ✨ 核心架构升级 (Key Architectural Upgrades)

* **🛡️ 100% 线程安全 (Thread Safety)**：引入 `DataHub` 全局数据交换中心，使用 FreeRTOS Mutex 与临界区保护 RC 意图数据，彻底杜绝数据撕裂。
* **⚡ 零浮点运算 (Zero-FPU Math)**：彻底重构了底层模拟量映射算法，使用 32 位整型乘除法（千分比映射）取代了所有的 `float` 运算，完美保护了 Cortex-M3 (STM32F103) 的核心算力，为 1kHz FOC 控制环路让路。
* **🚫 硬件抽象隔离 (HAL Decoupling)**：剥离了原有的 `HAL_UART_Receive_IT` 单字节中断，重构为基于 DMA 环形缓冲区 (RingBuffer) 的非阻塞轮询读取，极大降低了 CPU 中断负载。
* **🚨 硬件级安全降级 (Failsafe)**：L4 业务层实现主动失联检测。当侦测到遥控器关机或信号丢失超 100ms 时，系统将自动拦截脏数据，并向 DataHub 强制下发 `ESTOP_ACTIVE` (急停) 和 `ARM_CMD_HOLD`，防止机械臂暴走。

## 🛠️ 如何测试 (How to Test)

已包含完整的层级穿透测试模块 `test_rc_console`。

**启动步骤：**

1. 在 CubeMX 中配置 USART1 为 `Asynchronous`, `420000 bps`, 并开启 RX 的 `Circular DMA`。
2. 在 `main.c` 启动调度器前调用 `TestRcConsole_Init();`
3. 创建一个测试任务，在其循环内以 20ms 的周期调用 `TestRcConsole_TaskLoop();`
4. 连接串口终端，使用内置菜单进行逐层功能验证：
* `[r]` L2：测试通信链路与原始通道值
* `[s]` L3：测试长短按与急停逻辑判断
* `[d]` L4：测试 DataHub 的互斥锁读取及失控保护 (关掉遥控器电源触发)
* `[m]` SYS：进行 RC 控制指令与底层驱动 `DrvBldc_SetDuties` 的全链路联调。
